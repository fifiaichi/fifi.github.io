项目背景

岭南花市网站是本人的毕业设计项目，基于 PHP + MySQL 开发。在测试阶段，需要为商品表构造大量测试数据，以验证前台展示、分类筛选、价格显示等功能的正确性。

挑战：手工在后台录入 20 个商品信息（含名称、分类、价格、库存、描述、图片路径）耗时且容易出错，且无法保证数据格式统一。

解决思路：整理标准 CSV 文件，使用 AI 辅助编写 PHP 批量导入脚本，导入后用 Python 脚本验证数据完整性与业务逻辑正确性。

PHP 批量导入脚本（AI 辅助编写）

使用 Trae 生成基础框架，人工调整字段映射（type 对应分类字段、picture 路径拼接为 images/ 前缀）。 

// 数据库配置（huahua 库）
$host = 'localhost';
$dbname = 'huahua';
$user = 'root';
$pass = '';

// 连接数据库
$conn = mysqli_connect($host, $user, $pass, $dbname);
mysqli_set_charset($conn, 'utf8mb4');

// 读取 CSV
$csv = fopen('goods_batch.csv', 'r');
fgetcsv($csv); // 跳过表头

$success = 0;
$fail = 0;

while (($row = fgetcsv($csv)) !== false) {
    $name = mysqli_real_escape_string($conn, $row[0]);
    $type = mysqli_real_escape_string($conn, $row[1]);
    $price = (float)$row[2];
    $old_price = (float)$row[3];
    $stock = (int)$row[4];
    $description = mysqli_real_escape_string($conn, $row[5]);
    $picture = 'images/' . trim($row[6]);

    $sql = "INSERT INTO goods 
        (goods_name, type, price, old_price, description, picture) 
        VALUES ('$name', '$type', $price, $old_price, 
                '$description', '$picture')";

    if (mysqli_query($conn, $sql)) {
        $success++;
    } else {
        $fail++;
    }
}

echo "导入完成！成功：{$success} 个，失败：{$fail} 个";

Python 数据验证脚本

导入后使用 Python + pymysql 编写验证脚本，检查数据量一致性与业务规则（现价必须低于原价）。 

import pymysql
import csv

# 验证 1：数据库记录数 vs CSV 行数
def test_data_count():
    conn = pymysql.connect(
        host='localhost', user='root',
        password='', database='huahua'
    )
    cursor = conn.cursor()

    cursor.execute("SELECT COUNT(*) FROM goods")
    db_count = cursor.fetchone()[0]

    with open('goods_batch.csv', 'r', encoding='utf-8') as f:
        csv_count = len(f.readlines()) - 1

    assert db_count == csv_count
    print(f"✅ 数据量验证通过：{db_count} 条")

# 验证 2：价格逻辑（现价 < 原价）
def test_price_logic():
    cursor.execute("""
        SELECT goods_name, price, old_price 
        FROM goods 
        WHERE price >= old_price
    """)
    invalid = cursor.fetchall()
    assert len(invalid) == 0
    print("✅ 价格逻辑验证通过")

if __name__ == "__main__":
    test_data_count()
    test_price_logic()

代码审查与问题发现

在回顾导入脚本时，从测试视角发现了以下潜在风险。这也是我希望从事测试工作的原因：懂开发逻辑，更能从用户和边界角度发现问题。

ID 问题描述 风险等级 改进方向
ISS-01 SQL 语句直接拼接字符串，存在 SQL 注入风险 High 应使用预处理语句（Prepared Statements）
ISS-02 无事务控制，部分失败时数据库可能残留脏数据 Mid 应添加 BEGIN TRANSACTION / ROLLBACK
ISS-03 缺少重复数据校验，重复导入会产生重复记录 Low 导入前检查 goods_name 唯一性
